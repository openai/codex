# 贡献者许可协议（CLA）概览

> 本文是 `docs/CLA.md` 的中文说明，**仅供参考，不具有法律效力**。如有歧义，均以英文原文为准。

当你向本仓库提交贡献（例如通过 Pull Request）时，可能需要同意贡献者许可协议（Contributor License Agreement, CLA）。

## CLA 的目的

- 明确你对贡献代码拥有相应权利，可以授予给项目。
- 允许项目在既定开源许可证（例如 Apache 2.0）下继续分发和使用你的贡献。
- 保护贡献者和维护者，避免版权与授权方面的法律风险。

## 你通常需要确认的事情

具体条款请阅读英文原文，一般包括（但不限于）：

- 你是贡献代码的原始作者，或拥有足够的授权。
- 你授予项目在指定许可证下使用、修改、再分发你贡献的权利。
- 你不会故意提交有版权争议或违反他人权利的内容。

## 如何签署 / 同意 CLA

- 有些项目会使用自动化 CLA Bot，在第一次提交 PR 时引导你在线确认。
- 也有项目要求通过单独页面或文档签署。

请以本仓库实际工作流和 `docs/CLA.md` 中的说明为准。
