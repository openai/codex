## 🔄 今後の運用方針

### 1. 公式リポジトリとの同期

```bash
# 公式の変更を取得
git fetch upstream
git merge upstream/main

# zapabob独自機能は zapabob/ 配下で管理
# 競合が発生しにくい構造
```

### 2. 新規独自ファイルの配置

今後、zapabob独自のファイルを追加する際は:

- **ドキュメント** → `zapabob/docs/`
- **スクリプト** → `zapabob/scripts/`
- **拡張機能** → `zapabob/extensions/`
- **SDK** → `zapabob/sdk/`
- **レポート** → `zapabob/reports/`

### 3. コア機能の拡張

Rust実装（`codex-rs/`）内の独自機能:
- `mcp-server/` - MCP Server実装
- `supervisor/` - サブエージェント管理
- `deep-research/` - Deep Research実装

これらは公式にPR提出する可能性があるため、`codex-rs/` 配下に維持。

### 4. 実装ログの継続

`_docs/` ディレクトリは引き続き実装ログ用として使用:
- 日付_機能名.md 形式で保存
- エージェントとの対話履歴を記録
