# 零数据保留（Zero Data Retention, ZDR）

> 本文是 `docs/zdr.md` 的中文概览版本，描述 Zero Data Retention 模式的含义和注意事项。

Zero Data Retention（零数据保留）模式的目标是让你的请求和数据**不会被持久化用于模型训练或其他长期用途**（具体行为以官方政策和服务实现为准）。

## 一般含义

在启用 ZDR 时，通常意味着：

- 请求内容不会被用来训练或改进模型。
- 服务方不会长期存储这些数据。

但仍可能需要在短时间内缓存数据以：

- 完成当前请求。
- 支持必要的安全审计与滥用检测。

## 在 Codex 中的使用

- 当你在配置或账户中启用 ZDR 相关选项时，Codex 会尽量遵循该模式。
- 具体是否生效取决于：
  - 你所使用的 plan / contract。
  - 所选 provider 是否支持 ZDR。

详细条款和技术细节请以：

- 官方产品文档与隐私政策。
- 英文原文 `docs/zdr.md` 为准。
