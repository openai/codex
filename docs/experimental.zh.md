# 实验性功能（Experimental）

> 本文是 `docs/experimental.md` 的中文概览版本，主要说明实验性特性的一般原则。具体列表与状态以英文原文为准。

Codex 在快速迭代中会引入一些“实验性”功能，这类功能通常具有：

- 接口或行为尚未稳定，未来可能有破坏性变更。
- 主要面向早期尝鲜用户和贡献者，用于收集反馈。

## 使用实验性功能时的建议

- 在个人环境或试验分支中开启，避免直接影响生产环境。
- 注意阅读：
  - 对应文档的限制与注意事项。
  - Issue / PR 中的讨论。
- 当你发现问题或有改进建议时，欢迎：
  - 提交 Issue 描述使用场景与问题。
  - 提交 PR 改进实现或文档。

## 常见的实验性领域

具体功能随时间变化，通常包括：

- 新的模型 provider 或对话模式。
- 新的 sandbox 策略或平台支持。
- 更智能的上下文管理、文件搜索或 MCP 集成方式等。

最新列表和状态请查看英文文档 `docs/experimental.md` 和仓库中的 Issue/PR。
