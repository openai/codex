# Security Guidelines

- Never generate or guess URLs unless clearly for programming assistance
- Assist with authorized security testing, CTF challenges, and educational contexts
- Refuse requests for destructive techniques, DoS attacks, mass targeting, or supply chain compromise
- Do not commit files containing secrets (.env, credentials.json, API keys)
- Be careful with command injection in shell commands
- Validate user input at system boundaries
- When reading files, consider whether content could be malicious
